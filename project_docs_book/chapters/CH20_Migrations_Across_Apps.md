```markdown
# Chapter 20 — Migrations Across Apps

## Learning Goals
- Understand schema evolution order across all domain apps.
- Identify how later migrations align model updates with database structure.

## Reference Files
- `users/migrations/*`
- `hierarchy/migrations/*`
- `catalog/migrations/*`
- `inventory/migrations/*`
- `actions/migrations/*`
- `audit/migrations/*`

## Migration Timeline by App

## 1) Users
- `0001_initial.py`: custom user table with role field and auth relations.
- `0002_alter_user_role.py`: adds `CENTRAL_PROCUREMENT_STORE` role choice.
- `0003_user_office.py`: adds optional FK to `hierarchy.Office`.

## 2) Hierarchy
- `0001_initial.py`: creates `Office` with tree link (`parent_office`) and indexes.
- `0002_alter_office_level.py`: corrects level choice from `PROVENCIAL` to `PROVINCIAL`.

## 3) Catalog
- `0001_initial.py`: creates `Category` and `CustomFieldDefinition` with unique constraint.
- `0002_customfielddefinition_is_unique_and_more.py`:
  - adds `is_unique`,
  - adds `select_options` JSON,
  - extends `field_type` with `FILE`.

## 4) Inventory
- `0001_initial.py`: creates `InventoryItem`, `FixedAsset`, `ConsumableStock`.
- `0002_consumablestock_initial_quantity_and_more.py`:
  - adds stock thresholds/flags,
  - adds fixed-asset serial/warranty/invoice fields.
- `0003_consumablestocktransaction_inventoryitem_amount_and_more.py`:
  - creates `ConsumableStockTransaction`,
  - expands `InventoryItem` with item identity, pricing, documents, metadata, status extension, and indexes,
  - adds transaction links to users.

## 5) Actions
- `0001_initial.py`: creates `ItemAssignment` with constraints and indexes.
- `0002_itemassignment_assign_till.py`: adds `assign_till`.

## 6) Audit
- `0001_initial.py`: creates `InventoryAuditLog` with indexes and optional actor link.

## Observed Pattern
- Initial migrations establish core entities.
- Follow-up migrations add role choices, stronger workflow fields, and reporting-ready metadata.
- Dependencies correctly bridge cross-app relations (users/hierarchy/inventory/catalog).

## Chapter 20 Outcome
You now understand how the schema matured from core tables into workflow-rich structures across users, hierarchy, catalog, inventory, actions, and audit.
```
